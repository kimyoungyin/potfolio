## Design Context

### Users
- 채용 담당자(리크루터/엔지니어링 매니저)가 포트폴리오를 빠르게 스캔하며, 지원자의 역량과 실전 경험을 신뢰할 수 있는지 판단한다.

### Brand Personality
- 차분함: 불필요한 자극 없이 정보가 잘 읽히는 인터페이스
- 신뢰감: 일관된 디자인 시스템 사용, 명확한 계층 구조, 실수 적은 UI
- 기술적: 구현 디테일(구조/컴포넌트/접근성/토큰 기반)에서 “준비된 개발자”의 인상을 주는 톤
- 감정 목표: “이 사람이 실무에서 바로 기여할 수 있겠다”는 확신

### Aesthetic Direction
- 톤: 절제된 모던, 정돈된 타이포그래피, 토큰 기반 색/간격, 과한 장식 최소화
- 테마: 라이트/다크 모두에서 동일한 계층과 가독성을 유지
- 레퍼런스: 별도 선호 없음(대신 현재 프로젝트의 shadcn 스타일/OKLCH 토큰을 중심으로 정규화)

### Design Principles
- 스캔 가능성 우선: 섹션 제목/헤딩/버튼/링크가 빠르게 위치를 잡아야 한다.
- 토큰 기반 일관성: 색/반경/테두리/폰트/간격은 기존 디자인 토큰/유틸 클래스를 우선 사용한다.
- 타이포 계층 엄수: 크기·굵기·줄높이로 정보 우선순위를 명확히 한다.
- 신뢰감 있는 레이아웃: 불필요한 래핑/중복 스타일을 줄이고 컴포넌트 패턴을 유지한다.
- 접근성 기본값: 대비, 포커스, 상호작용 피드백, `prefers-reduced-motion` 고려를 지킨다.

## Design Context

### Users

- 채용 담당자(리크루터/엔지니어링 매니저)
- 사용자는 포트폴리오를 통해 `프로젝트 탐색`을 빠르게 수행하며, 기술적 결정과 실제 임팩트를 확인하고 싶어 함

### Brand Personality

- 차분함(calmbut)과 신뢰감(trust)
- 기술적 디테일 중심(사용자 표현: `기숙적`)

### Aesthetic Direction

- 과장된 장식보다 정보의 계층과 리듬을 우선하는 정돈된 인터페이스
- 섹션 단위의 일관된 spacing, `shadcn/ui` 스타일(OKLCH 기반 토큰), `max-w-5xl` 컨테이너 패턴 유지
- 레퍼런스는 없으므로 “generic AI 템플릿처럼 보이지 않게” 보이는 반복 패턴은 피하고, 기존 컴포넌트/토큰을 재사용해 완성도를 높임

### Design Principles

1. 일관성 우선: `components`에 정의된 UI 패턴과 shadcn 토큰을 그대로 사용하고, `app/page.tsx` 같은 조합 레이어는 최소한으로 유지
2. 가독성/계층: 섹션 제목-본문-액션의 위계를 명확히 하고, 폰트 크기/라인하이트/색 대비를 기존 토큰에 맞춤
3. 상호작용의 신뢰: hover/focus 상태가 예측 가능하게 동작하도록 현재 사용 중인 `Button/Badge` 등 컴포넌트의 variants를 준수
4. 모션 절제: 기존의 `animate-*` 클래스와 reduced-motion 고려를 유지(불필요한 과장 애니메이션은 추가하지 않음)
5. 접근성 포함: 링크에는 의미가 전달되도록 레이블을 유지하고, 버튼/인터랙티브 요소에는 스크린리더 접근성을 보장
